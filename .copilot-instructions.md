## Core Concepts

### Two-Phase Pattern

pyMyriad separates **construction** (defining what to analyze) from **execution** (running on data):

1. **Construction**: Build trees using `AnalysisTree()`, `.split_by()`, `.analyze_by()`
2. **Execution**: Run with `.run(df)` to get results

### Key Classes

**Construction Side** (analysis_tree.py):
- `AnalysisTree` (subclass of `list`) - Root container
- `SplitNode` (subclass of `list`) - Defines data splits/stratification
- `AnalysisNode` - Defines computations
- `CrossAnalysisNode` - Cross-level comparisons

**Results Side** (data_tree.py):
- `DataTree` (subclass of `dict`) - Root result container
- `SplitDataNode` (subclass of `dict`) - Split results
- `LvlDataNode` (subclass of `dict`) - Level-specific results
- `DataNode` - Leaf node with computed values

**Important**: AnalysisTree and SplitNode are lists, DataTree and SplitDataNode are dicts. This is intentional.

## Code Patterns

### Building Analysis Trees

Use method chaining for tree construction:

```python
tree = (AnalysisTree()
    .split_by('df.Gender', label='gender')
    .split_by('df.Country', label='country')
    .analyze_by(
        mean_income=lambda df: np.mean(df.Income),
        count=lambda df: len(df)
    ))
```

**Key methods**:
- `.split_by(expr, **kwexpr, label=None)` - Add split at leaf nodes
- `.analyze_by(**analysis)` - Add analysis at leaf nodes (terminates branch)
- `.summarize_by(**analysis)` - Add analysis without terminating
- `.split_at_by(path, expr, **kwexpr, label=None)` - Add split at specific location
- `.cross_analyze_by(**analysis, ref_lvl)` - Compare against reference level

### Expression Evaluation

Support **both** lambda functions and string expressions:

**Lambda (recommended for code)**:
```python
.analyze_by(
    mean=lambda df: np.mean(df.Income),
    count=lambda df: len(df)
)
```

**String expressions (useful for config-driven analysis)**:
```python
.analyze_by(
    mean="np.mean(df.Income)",
    count="len(df)"
)
```

**Implementation**: Use `scope_eval(expr, df, environ)` from utils.py
- If `expr` is callable: Call it with `df`
- If `expr` is string: Use `eval()` with `df`, `np`, `pd` in scope

**For cross-level comparisons**: Use `scope_cross_eval(expr, df, ref_df, environ)`
- Provides both `df` (current group) and `ref_df` (reference group)

### Environment Management

Default environment auto-imports numpy and pandas:

```python
# Set global default
AnalysisTree.set_default_environ({'np': np, 'pd': pd})

# Or per-run
result = tree.run(df, environ={'np': np, 'pd': pd, 'custom_func': my_func})
```

### Working with Results

Convert DataTree to DataFrame for further processing:

```python
from pyMyriad import simple_table, gt_table, forest_plot

# Basic DataFrame
df_result = simple_table(result, by='df.Gender')

# Formatted HTML table
html = gt_table(result, title="Analysis Results")

# Visualization
forest_plot(result, x='mean', x_err='std')
```

## Testing Patterns

### Test DataFrame Setup

Use **simple DataFrames** (2-6 rows) for tests:

```python
df = pd.DataFrame({
    'A': [10, 20, 30, 40],
    'B': [1, 2, 1, 2]
})
```

### Test Both Expression Forms

Every test should verify both lambda and string expressions work:

```python
def test_analysis_with_lambda():
    tree = AnalysisTree().analyze_by(mean=lambda df: np.mean(df.A))
    result = tree.run(df)
    assert 'mean' in result['analysis'].summary

def test_analysis_with_string():
    tree = AnalysisTree().analyze_by(mean="np.mean(df.A)")
    result = tree.run(df)
    assert 'mean' in result['analysis'].summary
```

### Test Structure Pattern

1. Create simple DataFrame
2. Build tree (test construction)
3. Run tree (test execution)
4. Assert on result structure and values

```python
def test_split_and_analyze():
    # Setup
    df = pd.DataFrame({'A': [1, 2, 3], 'B': ['X', 'Y', 'X']})
    
    # Construction
    tree = AnalysisTree().split_by('df.B').analyze_by(
        sum_a=lambda df: sum(df.A)
    )
    
    # Execution
    result = tree.run(df)
    
    # Assertions
    assert isinstance(result, DataTree)
    assert 'df.B' in result
    assert 'X' in result['df.B']
    assert result['df.B']['X']['analysis'].summary['sum_a'] == 4
```

### Performance Tests

Mark performance tests to skip by default:

```python
@pytest.mark.performance
def test_large_dataset():
    # Performance test code
    pass
```

Run with: `pytest -m performance`

## Common Function Signatures

```python
# Tree construction
AnalysisTree(id: str = None)
SplitNode(*args, expr: str = None, kwexpr: dict = None, label: str = None)
AnalysisNode(**analysis, termination: bool = True)

# Tree execution
AnalysisTree.run(data: pd.DataFrame, environ: dict = None) -> DataTree
SplitNode.run(data: pd.DataFrame, environ: dict = None, id: str = None, _N: int = None) -> SplitDataNode
AnalysisNode.run(data: pd.DataFrame, environ: dict = None, id: str = None, _N: int = None) -> DataNode

# Expression evaluation
scope_eval(expr: Union[str, Callable], df: pd.DataFrame, environ: dict = None) -> Any
scope_cross_eval(expr: Union[str, Callable], df: pd.DataFrame, ref_df: pd.DataFrame, environ: dict = None) -> Any

# Table generation

# Plotting
forest_plot(dtree: DataTree, x: str = "x", x_err: str = "err", **kwargs)
distribution_plot(dtree: DataTree, x: str, type: str = "scatter", **kwargs)

```

## Module Responsibilities

- **analysis_tree.py**: Tree construction, node classes, tree building methods, run() execution logic
- **data_tree.py**: Result data structures, pretty-printing
- **utils.py**: Expression evaluation (scope_eval, scope_cross_eval), string conversion, environment management
- **tabular.py**: Flattening DataTree to DataFrame, formatting statistics
- **listing.py**: User-facing table generation functions (simple_table, gt_table, cascade_table)
- **plots.py**: User-facing plotting functions (forest_plot, distribution_plot)
- **cli.py**: Command-line interface (minimal)

## Naming Conventions

- **Node classes**: Capitalized (AnalysisTree, SplitNode, DataNode)
- **Methods**: snake_case with descriptive names (split_by, analyze_by)
- **Parameters**: 
  - `df` for DataFrames passed to expressions
  - `expr` for single expression
  - `kwexpr` for dict of named expressions
  - `environ` for evaluation environment dict
  - `label` for human-readable labels
  - `_N` for entity count (with underscore)
- **Test functions**: Start with `test_` and describe what is being tested

## Important Notes

1. **List/Dict subclassing**: AnalysisTree and SplitNode inherit from `list`, DataTree and SplitDataNode inherit from `dict`. This is intentional for convenient indexing.

2. **Termination flag**: `AnalysisNode` has a `termination` attribute. If True (default), no further splits can be added after it. Use `summarize_by()` for non-terminating intermediate analyses.

3. **Path tracking**: Every node tracks its `path` and `depth` in the tree. This is critical for table generation and visualization.

4. **Flexible grouping**: `kwexpr` in SplitNode allows overlapping or non-exhaustive groups, unlike standard groupby operations.

5. **Expression safety**: String expressions use `eval()`. Recommend lambda functions for user-facing code, but support strings for config-driven workflows.

## When Suggesting Code

- **Prefer lambda functions** over string expressions in examples
- **Use method chaining** for tree construction
- **Include both construction and execution** in examples
- **Show result handling** (converting to tables/plots)
- **Test both simple and complex scenarios**
- **Include type hints** when adding new functions
- **Follow existing patterns** in the codebase

## Quick Example Template

```python
import pandas as pd
import numpy as np
from pyMyriad import AnalysisTree, simple_table

# Create data
df = pd.DataFrame({
    'Group': ['A', 'A', 'B', 'B'],
    'Value': [10, 20, 30, 40]
})

# Build analysis tree
tree = (AnalysisTree()
    .split_by('df.Group')
    .analyze_by(
        mean=lambda df: np.mean(df.Value),
        count=lambda df: len(df)
    ))

# Execute
result = tree.run(df)

# Convert to table
table = simple_table(result)
print(table)
```
